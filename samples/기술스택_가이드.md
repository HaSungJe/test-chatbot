# 테크노바 개발팀 기술 스택 가이드

## 개요

이 문서는 테크노바 개발팀에서 사용하는 기술 스택과 코드 컨벤션을 정리한 것입니다.
신규 입사자는 이 가이드를 숙지하고, 기존 코드 스타일과 일관성을 유지해 주세요.

---

## 프론트엔드

### 핵심 기술

- **Next.js 15 (App Router)** — 풀스택 프레임워크. 모든 신규 프로젝트는 App Router 방식 사용
- **React 19** — UI 라이브러리. React Compiler 활성화 상태
- **TypeScript** — 전면 도입. `any` 사용 금지, `strictNullChecks` 활성화
- **Tailwind CSS v4** — 유틸리티 퍼스트 스타일링

### 상태 관리

- 서버 상태: **TanStack Query (React Query)**. 캐시 전략은 `staleTime: 60_000` 기본값 사용
- 전역 상태: **Zustand**. 복잡한 전역 상태가 필요한 경우에만 사용
- 폼 상태: **React Hook Form + Zod**. 폼 검증은 반드시 Zod 스키마를 사용

### 코드 컨벤션

```
컴포넌트 파일명: PascalCase (예: UserProfile.tsx)
훅 파일명: camelCase, use 접두사 (예: useUserProfile.ts)
유틸 파일명: camelCase (예: formatDate.ts)
```

imports 순서:
1. React 관련
2. 외부 라이브러리
3. 내부 모듈 (@/로 시작하는 절대경로)
4. 타입 imports (import type)

## 백엔드

### 핵심 기술

- **NestJS** — 메인 API 서버 프레임워크
- **TypeORM** — ORM. 마이그레이션은 TypeORM CLI로 관리
- **PostgreSQL 16** — 메인 데이터베이스
- **Redis** — 세션, 캐시, 큐 처리

### API 설계 원칙

- RESTful 설계 준수
- 응답 형식 통일: `{ data: T, message: string, statusCode: number }`
- 에러 응답: `{ error: string, statusCode: number }`
- 페이지네이션: 커서 기반 페이지네이션 사용 (오프셋 방식 지양)

### 인증

- JWT Access Token (만료: 15분) + Refresh Token (만료: 7일)
- Access Token은 Authorization 헤더 Bearer 방식
- Refresh Token은 httpOnly 쿠키로 관리
- Access Token 만료 시 `/auth/refresh` 엔드포인트로 자동 갱신

## 인프라

### 배포 환경

- **컨테이너**: Docker + Docker Compose
- **오케스트레이션**: Kubernetes (EKS)
- **CI/CD**: GitHub Actions
- **이미지 레지스트리**: AWS ECR

### 브랜치 전략

```
main        — 프로덕션 배포 브랜치. 직접 push 금지, PR만 허용
develop     — 개발 통합 브랜치
feature/*   — 기능 개발 브랜치 (예: feature/user-auth)
fix/*       — 버그 수정 브랜치 (예: fix/login-crash)
hotfix/*    — 긴급 수정 브랜치 (프로덕션 이슈 대응)
```

### 모니터링

- **Sentry** — 에러 트래킹 (프론트/백엔드 모두 적용)
- **Grafana + Prometheus** — 서버 메트릭 모니터링
- **DataDog** — APM 및 로그 수집

## 개발 환경 세팅

### 필수 설치 도구

- Node.js 22 LTS
- Docker Desktop
- pnpm 9 (`npm install -g pnpm`)
- VS Code (권장 확장: ESLint, Prettier, GitLens)

### 로컬 실행

```bash
# 저장소 클론
git clone https://github.com/techknova/nova-erp.git
cd nova-erp

# 의존성 설치
pnpm install

# 환경변수 설정
cp .env.example .env.local

# DB 및 Redis 실행 (Docker)
docker-compose up -d db redis

# 마이그레이션
pnpm migration:run

# 개발 서버 실행
pnpm dev
```

### 코드 품질 도구

- **ESLint** — 코드 린팅. `pnpm lint` 로 실행
- **Prettier** — 코드 포맷팅. 저장 시 자동 포맷 설정 권장
- **Husky + lint-staged** — 커밋 전 자동 린팅/포맷팅

## 보안 가이드라인

- 환경변수에 시크릿 정보를 저장하고, 코드에 하드코딩 금지
- SQL 쿼리는 반드시 파라미터 바인딩 사용 (SQL Injection 방지)
- 사용자 입력값은 항상 서버 측에서 재검증
- 의존성 업데이트는 월 1회 정기적으로 진행 (`pnpm audit` 확인)
- 민감 정보(API 키, 비밀번호 등)를 로그에 출력 금지
